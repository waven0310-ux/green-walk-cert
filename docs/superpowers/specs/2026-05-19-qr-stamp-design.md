# QR 스탬프 완주 인증 시스템 설계

## 개요

기존 인증서 발급 시스템에 QR 스탬프 3개 수집 흐름을 추가한다.
참가자가 3개 체크포인트에서 QR을 순서대로 스캔하면 완주 인증서 발급 화면으로 진입한다.

---

## 화면 흐름

```
QR 스캔 (?cp=N&token=T)
  → screen-stamp (스탬프 수집 화면)
  → [3개 완료 시] screen-input (이름 입력, 기존 화면)
  → screen-cert (인증서, 기존 화면)
```

사이트 직접 접속 시 (QR 파라미터 없음):
- screen-stamp가 기본 랜딩 화면 (스탬프 0개 ~ 3개 상태 표시)
- 기존 screen-input은 더 이상 직접 접근 불가 (스탬프 3개 완료 후에만 진입)

---

## QR 코드 구조

각 체크포인트 QR은 아래 URL을 가리킨다:

```
CP 1: https://[site]/?cp=1&token=<TOKEN1>
CP 2: https://[site]/?cp=2&token=<TOKEN2>
CP 3: https://[site]/?cp=3&token=<TOKEN3>
```

- 토큰 3개는 JS에 하드코딩 (백엔드 불필요)
- 토큰 미일치 시 무시 (스탬프 미지급)

---

## 순서 강제 규칙

- CP2는 CP1 완료 후에만 인정
- CP3는 CP2 완료 후에만 인정
- 순서 어긴 QR 스캔 → 안내 메시지 표시, 스탬프 미지급
- 이미 찍은 QR 재스캔 → 현재 상태 그대로 유지 (중복 없음)

---

## 스탬프 상태 저장

- `localStorage` 키: `greenWalkStamps`
- 값: `[1, 2, 3]` 형태의 JSON 배열 (완료된 CP 번호 목록)
- 브라우저 닫아도 유지됨
- 완주 후 이미지 다운로드 버튼 클릭 시 localStorage 초기화 (재참가 대비)

---

## screen-stamp UI

- 행사 브랜딩 (2026 초록 발자국 남기기 / 패밀리 그린워크)
- 동그라미 3개 나란히 배치
  - 미수집: 점선 원 + 숫자
  - 수집 완료: 초록 채워진 원 + 캐릭터 도장 이미지 (동일 이미지 3개)
  - 스탬프 찍힐 때 애니메이션 (bounce/scale)
- 진행 텍스트: "N / 3 스탬프 수집"
- "완주 인증서 받기" 버튼: 3개 전 비활성, 3개 완료 시 활성화
- QR 스캔 직후 랜딩 시 해당 스탬프 찍히는 애니메이션 자동 실행

---

## 필요 에셋

| 에셋 | 설명 | 상태 |
|------|------|------|
| 캐릭터 도장 이미지 | PNG, 원형 투명배경 권장 | 사용자 제공 예정 |
| QR 코드 3개 | 인쇄용 PNG | 구현 후 생성 |

---

## 구현 범위

- `index.html` 단일 파일 수정 (신규 파일 없음)
- 추가 화면: `screen-stamp`
- 추가 로직: URL 파라미터 파싱, 토큰 검증, localStorage 관리
- 기존 `screen-input`, `screen-cert` 화면 변경 없음
- 완주 조건: 스탬프 3개 → "완주 인증서 받기" 버튼 → 기존 이름 입력 흐름 연결
